# 思考をまとめるチェックリスト（新規事業・サービス企画向け）

企画の検討・仮説検証・社内説明・意思決定の前後で、考えを整理するためのチェックリストです。  
該当するフェーズだけ使ってください。

---

## 使い方（フェーズ別）

| フェーズ | 重点セクション |
|---|---|
| アイデア検討 | 課題の解像度 → 市場・競合 → 目的・ゴール |
| 仮説検証 | 課題の解像度 → 仮説と検証設計 → 判断材料 |
| 社内説明・提案 | 目的・ゴール → 事業モデル → 論点整理 → 共有用サマリー |
| 意思決定 | 判断材料 → リスク・前提 → 次のアクション |

---

## 1. 課題の解像度

- [ ] **誰の**課題かを具体的に書いた（職種・年齢・状況など、1人の人物像で）
- [ ] **どんな状況で**その課題が起きるかを書いた（シーン・頻度・深刻度）
- [ ] 課題は「不便」か「痛点（放置できない問題）」か判断した
- [ ] ユーザーが**今どうやって解決しているか**（代替手段・我慢）を書いた
- [ ] その課題を**自分で体験・観察した**か、または**直接話を聞いた**か確認した

---

## 2. 市場・競合

- [ ] **市場規模**の概算（TAM / SAM / SOM）を調べた or 仮置きした
- [ ] **主要競合**を3つ以内で挙げ、それぞれの強み・弱みを書いた
- [ ] 競合と比べた**自分たちの差別化ポイント**を1文で書いた
- [ ] **参入タイミング**が早すぎ・遅すぎないか判断した（市場の熟成度）
- [ ] 競合がまだやっていない理由（参入障壁 or 見落とし）を考えた

---

## 3. 仮説と検証設計

- [ ] 企画の核にある**最大の仮説**（これが崩れると成立しない前提）を1文で書いた
- [ ] その仮説を**最小コストで検証する方法**を考えた（ユーザーインタビュー・プロトタイプ・MVP など）
- [ ] 検証で**何が確認できれば前進するか**（判断基準）を決めた
- [ ] 検証に必要な**期間・リソース・協力者**を書いた
- [ ] 仮説が外れた場合の**ピボット方向**を1つ考えた

---

## 4. 事業モデルの論点

- [ ] **誰に・何を・どう届けるか**（提供価値と届け方）を1文で書いた
- [ ] **誰がお金を払うか**（ユーザー ≠ 顧客のケースを確認）を整理した
- [ ] **収益の仕組み**（月額・従量・手数料・広告など）を書いた
- [ ] **主なコスト構造**（開発・人件費・獲得コストなど）を把握した
- [ ] **継続・拡大の仕組み**（リピート動機・ネットワーク効果・参入障壁）を考えた

---

## 5. 目的・ゴール

- [ ] **この企画の目的**を1文で書いた（なぜやるか）
- [ ] **成功の定義**を書いた（何が達成されればOKか）
- [ ] **期限・優先度**（高／中／低）を決めた
- [ ] **決めること**と**決めなくていいこと**を分けた
- [ ] **関係者が求めている成果**（上司・経営・チーム）を書いた

---

## 6. 論点整理

- [ ] 論点を**3つ以内**に絞った
- [ ] 各論点の**選択肢**（A / B / 現状維持 など）を書いた
- [ ] 各選択肢の**メリット・デメリット**を書いた
- [ ] **自分の推奨案**と、その理由を1文で書いた
- [ ] **想定される反対意見・懸念**を1つ以上書いた

---

## 7. 判断材料

- [ ] 判断に使える**データ・数値・ユーザーの声**を確認した
- [ ] **過去の事例**（社内外の類似企画・失敗例）を調べた
- [ ] **制約条件**（予算・工数・技術・契約・法務）を書いた
- [ ] **誰の判断が必要か**（自分で決める／承認が必要）を確認した
- [ ] **情報不足**なら、誰に・何を・いつまでに確認するか決めた

---

## 8. リスク・前提

- [ ] 判断の**前提**（〜であると仮定している）を書いた
- [ ] 前提が崩れたときの**影響**を書いた
- [ ] **想定リスク**（起きうる問題）を3つ以内で書いた
- [ ] リスクごとに**対策 or 回避策**を書いた
- [ ] **やらないリスク**（放置した場合の機会損失）も確認した

---

## 9. 次のアクション

- [ ] やることを**動詞で始まるタスク**に分解した（例：確認する／作成する／連絡する）
- [ ] **担当者**（自分／誰）を決めた
- [ ] **期限**（今日／今週／◯月◯日）を決めた
- [ ] **最初の1アクション**（次に手を付けること）を1つ選んだ

---

## 10. 共有用サマリー

社内説明・チャット・メール用に、以下を埋めてから共有する。

- [ ] **結論・提案**を1文で書いた
- [ ] **背景・課題・理由**を2〜3文で書いた
- [ ] **依頼・確認事項**（相手にしてほしいこと）を箇条書きにした
- [ ] **期限・次回更新**のタイミングを書いた
- [ ] 相手の立場（経営／現場／非専門家）に合わせて**言い換え**た

---
