# 思考をまとめるチェックリスト（業務用）

会議・提案・トラブル対応・意思決定の前後で、考えを整理するためのチェックリストです。  
該当するシーンだけ使ってください。

---

## 使い方（シーン別）

| シーン | 重点セクション |
|---|---|
| 会議前 | 状況整理 → 目的・ゴール → 共有用サマリー |
| 提案前 | 目的・ゴール → 論点整理 → 判断材料 → リスク・前提 |
| トラブル対応 | 状況整理 → 論点整理 → 次のアクション |
| 意思決定 | 判断材料 → リスク・前提 → 次のアクション |

---

## 1. 状況整理

- [ ] **何が起きているか**を1〜2文で書いた（事実のみ）
- [ ] **いつ・誰・どこ**が関係しているか整理した
- [ ] **現状の影響**（ユーザー／売上／納期／チーム）を書いた
- [ ] **事実**と**推測・意見**を分けて書いた
- [ ] まだ**確認できていないこと**をリストアップした

---

## 2. 目的・ゴール

- [ ] **この仕事の目的**を1文で書いた（なぜやるか）
- [ ] **成功の定義**を書いた（何が達成されればOKか）
- [ ] **期限・優先度**（高／中／低）を決めた
- [ ] **決めること**と**決めなくていいこと**を分けた
- [ ] **関係者が求めている成果**（上司・クライアント・チーム）を書いた

---

## 3. 論点整理

- [ ] 論点を**3つ以内**に絞った
- [ ] 各論点の**選択肢**（A / B / 現状維持 など）を書いた
- [ ] 各選択肢の**メリット・デメリット**を書いた
- [ ] **自分の推奨案**と、その理由を1文で書いた
- [ ] **反対意見・懸念**を1つ以上書いた

---

## 4. 判断材料

- [ ] 判断に使える**データ・ログ・数値**を確認した
- [ ] **過去の事例**（同様の対応・失敗）を思い出した／調べた
- [ ] **制約条件**（予算・工数・技術・契約・法務）を書いた
- [ ] **誰の判断が必要か**（自分で決められる／承認が必要）を確認した
- [ ] **情報不足**なら、誰に・何を・いつまでに確認するか決めた

---

## 5. リスク・前提

- [ ] 判断の**前提**（〜であると仮定している）を書いた
- [ ] 前提が崩れたときの**影響**を書いた
- [ ] **想定リスク**（起きうる問題）を3つ以内で書いた
- [ ] リスクごとに**対策 or 回避策**を書いた
- [ ] **やらないリスク**（放置した場合の損失）も確認した

---

## 6. 次のアクション

- [ ] やることを**動詞で始まるタスク**に分解した（例：確認する／作成する／連絡する）
- [ ] **担当者**（自分／誰）を決めた
- [ ] **期限**（今日／今週／◯月◯日）を決めた
- [ ] **最初の1アクション**（次に手を付けること）を1つ選んだ
- [ ] ブロック要因があれば、**誰にエスカレーションするか**決めた

---

## 7. 共有用サマリー

会議・チャット・メール用に、以下を埋めてから共有する。

- [ ] **結論・提案**を1文で書いた
- [ ] **背景・理由**を2〜3文で書いた
- [ ] **依頼・確認事項**（相手にしてほしいこと）を箇条書きにした
- [ ] **期限・次回更新**のタイミングを書いた
- [ ] 相手の立場（経営／開発／非エンジニア）に合わせて**言い換え**た

---

## 記入テンプレート

```
【シーン】会議前 / 提案前 / トラブル対応 / 意思決定

■ 状況（事実）
-

■ 目的・ゴール
-

■ 論点と推奨
-

■ 判断材料・根拠
-

■ リスク・前提
-

■ 次のアクション（担当・期限）
1.
2.

■ 共有用（1文サマリー）
-
```
